# Proof of Origin

We live in a moment where any image you see online might have been generated by an AI in seconds, and there is no reliable way to tell. Metadata gets stripped, screenshots lose their history, and a single re-save is enough to sever any connection to the original source. Proof of Origin is an attempt to fix that — a system that brands every AI-generated image with a set of invisible marks that survive compression, cropping, screenshots, and even being fed back into another AI model, and then permanently records the proof of that image's existence on a public blockchain so that anyone, anywhere, without trusting any server, can verify where it came from.

When a user generates an image through the app, three separate watermarks are woven into it simultaneously — one lives in the mathematical structure of the image's frequency patterns and survives even aggressive AI regeneration, one is a neural steganographic layer that survives JPEG compression and resizing, and one is a human-readable metadata signature attached to the file. At the same time, a zero-knowledge proof is generated that cryptographically binds the image to its origin without revealing the original data, and that proof is submitted to a smart contract on Base Sepolia where it is verified and stored forever. The full provenance record — what prompt was used, which model generated it, when, and every hash that identifies the image — is written permanently to Arweave so it can never disappear even if this server does.

Verifying an image is just as simple. You upload any version of it — the original, a screenshot, a crop, a JPEG someone shared three times — and the system tries to read the watermarks back out. If it finds them, it checks the blockchain record and tells you whether what you have is the exact original or a transformed derivative, how far it has drifted from the registered version, and whether the zero-knowledge proof on-chain confirms the image was honestly watermarked at the moment of creation. There is also a button that skips the backend entirely and queries the smart contract directly from the browser, so the verification can be trusted even by someone who does not trust us.

**Why this matters now.** The EU AI Act — which begins enforcement in 2025 — requires providers of AI-generated content to ensure their outputs are machine-detectable as synthetic. Existing approaches rely on fragile metadata that disappears after a single screenshot. Proof of Origin is the first system to combine frequency-domain watermarking (which survives AI regeneration), blockchain-verified zero-knowledge proofs, and permanent decentralized storage into a single pipeline that satisfies this requirement by design, not by convention. Regulators, journalists, and ordinary users do not need to trust the platform — the proof lives on a public chain that anyone can query.

**Innovation.** Three ideas in this project have not been combined before: (1) using DINOv2 semantic embeddings as a perceptual hash that detects derivatives even after img2img AI regeneration — something traditional pHash algorithms cannot do; (2) generating an ezkl zero-knowledge proof over the combined (watermark ID, image hash, perceptual hash) tuple and verifying it on-chain at registration time, so the blockchain itself vouches for the honest watermarking of the image rather than merely storing a hash someone submitted; (3) a two-phase on-chain registration pattern (placeholder → complete) that eliminates the crash window between image generation and blockchain anchoring, making the system reliable enough for production without any manual recovery.

**Secure architecture.** The system is built with multiple independent security layers so that no single point of failure breaks the guarantee. Watermarks use three different techniques with different threat models — an attacker must defeat frequency-domain steganography, neural pixel encoding, and metadata signing simultaneously to remove provenance. The ZK verifier contract is generated directly from the circuit definition by ezkl and deployed independently; even if this backend is compromised, an on-chain record with `proofVerifiedOnChain = true` cannot be fabricated retroactively. Private keys never leave the server environment; the blockchain wallet controls only the registration transactions on a testnet. The SQLite outbox with WAL mode ensures atomic writes so a server crash cannot produce a watermarked-but-unregistered image. Rate limiting, input validation, and no direct user control over wallet operations complete the perimeter.
